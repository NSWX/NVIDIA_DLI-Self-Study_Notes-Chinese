#### 具身智能的主要能力

**1. 自主导航**

自主导航是具身智能的主要能力之一。这使机器人能够在各种环境中独立移动，无需人工干预即可做出路径决策并避开障碍物。

**2. 物体操作**

物体操作是具身智能的另一个关键方面，包括：

- **抓取**：安全地握住不同形状和大小物体的能力
- **复杂操作**：执行精细任务，如组装组件或叠衣服

这些技能使机器人能够以密切模仿人类能力的方式与其环境交互。

#### 仿真与测试

NVIDIA Isaac Sim、NVIDIA Isaac Lab 或 Gazebo 等仿真环境帮助我们在将机器人部署到真实场景之前验证整个系统。我们还可以利用各种测试框架，如硬件在环（HIL）测试、软件在环（SIL）测试以及单元测试或系统级测试。

#### 开发框架

机器人开发通常依赖于专门的平台和框架。一个著名的例子是 **ROS（机器人操作系统）**，它为编写机器人软件提供了灵活的框架。也可以使用其他中间件操作系统。

### 🛠️ 硬件集成

硬件集成的目标是将各种硬件组件组合起来作为一个 cohesive 系统无缝运行，以确保传感器、执行器和控制器之间的可靠和高效交互。

#### 传感器

传感器对于环境感知、导航辅助和为控制系统提供反馈至关重要。常见类型包括：

| 传感器类型              | 功能描述 | 应用场景           |
| ----------------------- | -------- | ------------------ |
| **摄像头**              | 视觉输入 | 物体识别、导航     |
| **激光雷达（LIDAR）**   | 距离测量 | 3D 建图、避障      |
| **IMU（惯性测量单元）** | 方向测量 | 姿态估计、运动跟踪 |
| **GPS**                 | 位置跟踪 | 户外定位、导航     |

#### 执行器

执行器实现运动和操作，根据传感器反馈施加力和进行调整。执行器类型包括：

| 执行器类型           | 描述                       |
| -------------------- | -------------------------- |
| **电机**             | 直流电机和步进电机用于运动 |
| **伺服电机**         | 用于精确位置控制           |
| **气动和液压执行器** | 用于力的施加               |

#### 控制器

控制器处理传感器数据，执行控制算法，并促进组件之间的通信：

| 控制器类型     | 示例                  |
| -------------- | --------------------- |
| **微控制器**   | Arduino、PIC          |
| **单板计算机** | 树莓派、NVIDIA Jetson |
| **嵌入式系统** | 集成控制解决方案      |

### 🧠 感知（Perception）

在机器人世界中，感知对于使自主机器人理解其环境至关重要。就像人类使用视觉、听觉和触觉等多种感官来解释周围的世界一样，机器人依赖于**传感器融合**。这个过程整合来自多个传感器的数据，以创建环境的全面和准确表示。

#### 感知的关键组件

- **传感器融合**：结合来自各种传感器的数据，提供周围环境的整体视图
- **物体检测与识别**：识别和分类机器人环境中的物体
- **物体跟踪**：持续跟踪移动的物体，类似于我们的眼睛跟踪移动物体
- **场景理解**：解释环境的整体上下文，使机器人能够做出明智的决策

#### NVIDIA Isaac Perceptor

NVIDIA 开发了 **Isaac Perceptor**，这是一个专为移动机器人设计的基于摄像头的3D感知系统。该系统提供：

- **鲁棒里程计**：跟踪机器人在其环境中的移动
- **局部3D场景重建**：构建周围环境的三维地图以帮助自主导航

Isaac Perceptor 利用 **VSLAM（视觉同步定位与建图）** 等技术进行准确的里程计测量，以及 **NVBlox** 进行详细的3D重建。

### 🗺️ 定位与建图

定位和建图对于机器人有效导航和与其环境交互至关重要。想象一个机器人就像需要地图和指南针来探索陌生领域的旅行者。这些过程使机器人能够确定其位置并创建其周围环境的地图。

#### 定位的重要性

定位帮助机器人识别其在环境中的位置，使其能够做出明智的决策：

- **GPS（全球定位系统）**：主要用于户外提供位置数据
- **视觉里程计**：利用摄像头图像估计运动，将任务转化为计算问题
- **基于激光雷达的定位**：使用基于激光的传感器将当前位置与预先存在的地图匹配

#### 建图技术

- **SLAM（同步定位与建图）**：机器人同时构建地图并跟踪其在其中位置的技术
- **占据栅格**：将环境表示为栅格，每个单元格指示被占据的概率

### 🧭 导航与路径规划

在自主移动机器人中，导航和路径规划对于确保机器人能够高效安全地在其环境中移动非常重要。这涉及几个规划层，每层都有特定的重点。

#### 经典导航系统

**路由与任务规划**

路由和任务规划确定地图网络上的优选路线。通常使用 A*、D* 和 RRT（快速探索随机树）等算法。这类似于在十字路口获取方向——决定是直行还是转弯。

这通常单独使用是不够的，所以我们需要额外的规划层。这就是引入运动规划的原因。

**运动规划**

运动规划生成一系列运动，使机器人能够沿着规划的路径移动同时避开障碍物。处理决策和轨迹生成两个组件通常需要几秒钟。

- **决策**：专注于行为规划和交互，如决定是绕过托盘等障碍物还是让行于移动的叉车
- **轨迹生成**：涉及创建基于时间的轨迹，详细说明精确控制的速度、加速度和航向角

#### 避障

- **实时检测**：通过实时检测和避开障碍物确保功能安全
- **反应式导航**：使用传感器进行即时调整，当主自主系统未能检测到障碍物时作为备用

### 🎮 机器人控制系统

一旦机器人的轨迹和目标状态被定义，控制系统将这些信息转化为机器人执行器的命令。这确保机器人准确地遵循规划的路径。

#### 反馈控制

反馈控制根据传感器输入调整机器人的动作以实现期望的行为。

一种常见的反馈控制方法是 **PID 控制**（比例-积分-微分）。PID 控制是一种广泛使用的算法，通过持续调整运动来最小化期望位置和实际位置之间的误差。它通过比较期望轨迹状态与从定位数据获得的实际状态并进行实时校正来操作。

#### 轨迹跟踪

你可以使用轨迹跟踪来确保机器人准确地遵循规划的轨迹。

一种常用的方法是 **MPC**（模型预测控制）。MPC 是一种先进的控制策略，使用机器人行为的预测模型来优化其路径。MPC 比 PID 控制更复杂，通常用于人形机器人的精确轨迹跟踪和全身控制。

### 🚀 移动基础模型

移动基础模型充当机器人系统的中央"大脑"，处理鲁棒的视觉状态并生成世界建模和动作策略。该模型旨在跨各种机器人实体工作，包括自主移动机器人（AMRs）。

#### 主要特点

- **基于仿真的训练**：模型仅使用仿真数据进行训练，允许无需物理试验的高效开发
- **零样本部署**：无需额外训练即可成功部署在 Hubble Lab 等真实环境中，展示了其适应性
- **跨实体能力**：为 AMRs 开发的策略可转移到其他机器人，如人形机器人、四足机器人和太空叉车

### 🧩 VLM、LLM 和 RAG 在机器人中的应用

视觉语言模型（VLM）、大语言模型（LLM）和检索增强生成（RAG）的集成正在通过一种称为**偶发感知**的概念改变机器人技术。这种方法允许机器人构建和利用记忆来有效地导航和与其环境交互。

#### 主要特点

- **记忆构建**：机器人使用这些模型记住事件、位置和时间，使其能够在需要时返回特定位置
- **演示示例**：在最近的一次 NVIDIA 办公室演示中，一个机器人被指示找到一个"漂亮的景色"。它回忆起以前的观察，如绿色植物，并引导用户到该位置
- **任务执行**：机器人可以基于记忆执行任务。例如，它可以通过记住以前在哪里看到过薯片来在厨房中找到薯片

#### 技术组件

- **视频语言模型（VLA）**：每隔几秒捕获和描述视频数据，将这些信息存储在向量数据库中以供长期记忆
- **语音处理**：利用 NVIDIA 的 Riva 将语音转换为文本，增强交互能力
- **开放词汇学习**：允许机器人理解和响应新的多样化查询，无需预定义类别

#### 机遇

- **边缘部署**：直接在设备上运行模型可提高性能并减少对云的依赖，如 NVIDIA 的 VLA 演示所示
- **基准开发**：创建基准有助于跨场景比较算法，提高可重复性和性能评估
- **人机交互**：增强社交导航和理解人类期望可改善人类与机器人之间的协作
